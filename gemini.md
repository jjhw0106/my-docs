* 개발일지 작성 방법
  - 개발일지 md파일 작성은 C:\Users\jhw\workspace\my-backoffice\../개발일지/ 해당 경로에 적절한 디렉토리 하위에 작성한다.
  - 예를 들어, front 관련 개발일지는 /front 하위에 작성
  - 네이밍 규칙은 '251227_프론트'와 같이 날짜_프로젝트명 으로 하고 해당 파일의 양식을 참고하여 작성한다.

*  To-Do 작성 방법
  - To-Do.md 파일에 작성
  - 작성 전에, 최종적으로 작업 내용 확인해서 반영된 부분 확정하고 작성 시작
  - 존댓말이 아닌 명사형으로 끝나는 문장을 사용할 것
  - 맨 위에 작업한 날짜를 적고(추가하지말고 최종 날짜 기준으로 업데이트) 그 밑에 순서대로 작성
  - FE / BE에서 할 일을 나눠서 작성
  - 기본적으로 사용자가 말한 내용을 추가하고, 그 외 GEMINI가 판단했을 때 해야한다고 생각하는 내용을 추가
  - 작업 내용 중 해결한 부분은 하위 항목에 가운제 취소선 그어서 체크할것
  - 전부 완료된 건 [] 안에 체크도 추가
